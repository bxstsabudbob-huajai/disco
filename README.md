# Discord Bot: ระบบรับยศ + Ticket + กันสแปม

พัฒนาโดย **@nongbest**

บอทนี้มี 3 ระบบหลัก ตามที่กำหนด:
1. **ระบบรับยศ** — กดปุ่มรับยศได้เอง (ใส่รูปในกล่องได้), เด้ง DM แจ้งเตือนหลังรับ/ถอดยศเสร็จ, คำสั่ง `/giverole` ใช้ได้เฉพาะแอดมิน
2. **ระบบ Ticket** — เลือกประเภทได้ 3 อย่างผ่านเมนู, แนบรูปในตั๋วได้, บันทึก transcript เป็นรูปภาพ (แบบกล่อง) ลง log และส่งกลับไปหาผู้เปิดตั๋วทาง DM พร้อมระบุว่าใครเปิด/ใครรับเรื่อง/ใครปิด, คำสั่ง `/ticket` ใช้ได้เฉพาะแอดมิน
3. **ระบบกันสแปม** — ตรวจจับข้อความรัว (flood), แท็กสมาชิก/โรลจำนวนมาก, ลิงก์เชิญ Discord แล้วลบข้อความ + Timeout อัตโนมัติ พร้อมบันทึกทุกอย่างลง log

UI ทั้งหมดแสดงผลเป็น **กล่อง (Embed)** ตามที่กำหนด

## การติดตั้ง

### 1. เตรียมบอทใน Discord Developer Portal
1. ไปที่ https://discord.com/developers/applications แล้วสร้าง Application ใหม่
2. ไปที่แท็บ **Bot** > กด **Reset Token** เพื่อคัดลอก Token
3. เปิด **Privileged Gateway Intents** ให้ครบ: `SERVER MEMBERS INTENT`, `MESSAGE CONTENT INTENT`
4. ไปที่แท็บ **OAuth2 > URL Generator** เลือก scope `bot` และ `applications.commands`
   - Permissions ที่ต้องติ๊ก: `Manage Roles`, `Manage Channels`, `Moderate Members` (สำหรับ timeout), `Send Messages`, `Attach Files`, `Read Message History`, `View Channels`, `Embed Links`
5. ใช้ลิงก์ที่ได้เชิญบอทเข้าเซิร์ฟเวอร์

> **สำคัญ**: ยศของบอทในเซิร์ฟเวอร์ต้อง **อยู่สูงกว่า** ยศทั้งหมดที่ต้องการให้บอทมอบ/ถอด ไม่งั้นบอทจะมอบยศไม่ได้

### 2. ติดตั้งโปรแกรม
ต้องมี [Node.js](https://nodejs.org) เวอร์ชัน 18 ขึ้นไป

```bash
npm install
```

### 3. ตั้งค่าไฟล์ .env
คัดลอก `.env.example` เป็น `.env` แล้วกรอกข้อมูล:

```
DISCORD_TOKEN=โทเค็นบอทของคุณ
CLIENT_ID=Application ID ของบอท (หน้า General Information)
GUILD_ID=Server ID ของเซิร์ฟเวอร์ (คลิกขวาที่ไอคอนเซิร์ฟเวอร์ > Copy Server ID)
```

> ถ้าใส่ `GUILD_ID` คำสั่งจะอัปเดตทันทีเฉพาะเซิร์ฟเวอร์นั้น (แนะนำตอนพัฒนา/ทดสอบ)
> ถ้าไม่ใส่ คำสั่งจะเป็น Global ใช้เวลาประมาณ 1 ชั่วโมงถึงจะขึ้น

### 4. ตั้งค่าไฟล์ config.json
แก้ไขไฟล์ `config.json` ให้ตรงกับเซิร์ฟเวอร์ของคุณ:

| ฟิลด์ | คำอธิบาย |
|---|---|
| `logChannelId` | ID ของช่องสำหรับบันทึก log ทั้งหมด (รับยศ, ticket, กันสแปม) |
| `adminRoleId` | ID ยศแอดมิน/ทีมงานที่ให้สิทธิ์รับ/ปิด ticket ได้ (นอกเหนือจาก Administrator) |
| `roles` | รายการยศที่กดรับได้ผ่านปุ่ม (ใส่ `roleId` จริงจากเซิร์ฟเวอร์) |
| `ticketCategories` | ประเภท ticket 3 อย่าง พร้อม `categoryId` (หมวดหมู่ช่องที่จะสร้าง ticket ไว้ข้างใน ถ้าไม่ใส่จะสร้างนอกหมวดหมู่) |
| `antiSpam` | ตั้งค่าความไวของระบบกันสแปม |

วิธีดู ID: เปิด **Developer Mode** ใน Discord (Settings > Advanced) แล้วคลิกขวาที่ยศ/ช่อง/หมวดหมู่ > Copy ID

### 5. ลงทะเบียนคำสั่ง Slash Command

```bash
npm run deploy
```

### 6. เริ่มรันบอท

```bash
npm start
```

## วิธีใช้งานระบบรับยศ

- แอดมินพิมพ์ `/giverole setup` ในช่องที่ต้องการ (ใส่หัวข้อ/คำอธิบาย/รูปภาพได้ตามต้องการ) บอทจะโพสต์กล่องพร้อมปุ่มยศ
- สมาชิกกดปุ่มเพื่อรับยศ (กดซ้ำอีกครั้งเพื่อถอดยศ) ได้รับ DM แจ้งเตือนทันที
- แอดมินสามารถมอบยศให้สมาชิกโดยตรงด้วย `/giverole give สมาชิก:@user ยศ:@role`
- ทุกการรับ/ถอด/มอบยศจะถูกบันทึกลงช่อง log

## วิธีใช้งานระบบ Ticket

- แอดมินพิมพ์ `/ticket setup` ในช่องที่ต้องการ บอทโพสต์กล่องพร้อมเมนูให้เลือก 3 ประเภท
- สมาชิกเลือกประเภทจากเมนู บอทสร้างช่องส่วนตัวให้อัตโนมัติ (มองเห็นเฉพาะเจ้าของ+แอดมิน)
- ในช่อง ticket มีปุ่ม **รับเรื่อง** (เฉพาะแอดมิน) และ **ปิด Ticket** (แอดมินหรือเจ้าของตั๋ว)
- เมื่อปิด ticket บอทจะบันทึก transcript ทั้งหมด (ข้อความ + รายชื่อไฟล์แนบในตั๋ว) เป็น **รูปภาพสไตล์กล่อง (.png)** ส่งไปที่ log และ **DM กลับไปหาผู้เปิดตั๋วโดยอัตโนมัติ** พร้อมสรุปว่า **ใครเปิด / ใครรับเรื่อง / ใครปิด** (ถ้าบทสนทนายาว จะแบ่งเป็นหลายรูปภาพ)
- ช่อง ticket จะถูกลบอัตโนมัติ 5 วินาทีหลังปิด (transcript ถูกเก็บไว้ใน log และส่งให้ผู้เปิดแล้ว)

## ระบบกันสแปม

ทำงานอัตโนมัติทุกข้อความในเซิร์ฟเวอร์ (ยกเว้นแอดมิน/ยศที่อยู่ใน whitelist) ตรวจจับ:
- ส่งข้อความถี่เกินกำหนด (ตั้งค่าได้ที่ `maxMessages` / `timeWindowSeconds`)
- แท็กสมาชิก/โรลเกินจำนวนที่กำหนด (`maxMentions`)
- แชร์ลิงก์เชิญ Discord (`deleteInvites`)

เมื่อตรวจพบ: ลบข้อความทันที + Timeout สมาชิกตามเวลาที่ตั้งไว้ (`timeoutMinutes`) + บันทึกลง log ทุกครั้ง

## โครงสร้างโปรเจกต์

```
discord-bot/
├── index.js              # ไฟล์หลัก เริ่มรันบอท
├── deploy-commands.js     # ลงทะเบียน slash command
├── config.json            # ตั้งค่ายศ/ticket/กันสแปม
├── .env.example            # ตัวอย่างไฟล์ token
├── commands/
│   ├── giverole.js        # /giverole (admin only)
│   └── ticket.js           # /ticket (admin only)
├── events/
│   ├── ready.js             # เมื่อบอทออนไลน์
│   ├── interactionCreate.js  # จัดการปุ่ม/เมนู/คำสั่งทั้งหมด
│   └── messageCreate.js       # ระบบกันสแปม
├── utils/
│   ├── logger.js            # ส่ง log แบบ embed
│   ├── antispam.js           # ตรรกะตรวจจับสแปม
│   ├── transcript.js          # สร้างรูปภาพบันทึก ticket (สไตล์กล่อง)
│   └── storage.js             # อ่าน/เขียนข้อมูล JSON
└── data/                    # เก็บข้อมูล ticket (สร้างอัตโนมัติ)
```

## หมายเหตุ

- ข้อมูล ticket เก็บเป็นไฟล์ JSON ในโฟลเดอร์ `data/` (เหมาะกับเซิร์ฟเวอร์ขนาดเล็ก-กลาง) หากต้องการสเกลใหญ่ขึ้นแนะนำเปลี่ยนไปใช้ฐานข้อมูลจริง เช่น SQLite/MongoDB
- ควรรันบอทบนเครื่อง/โฮสต์ที่เปิดทิ้งไว้ตลอดเวลา เช่น VPS หรือบริการโฮสต์บอท Discord
